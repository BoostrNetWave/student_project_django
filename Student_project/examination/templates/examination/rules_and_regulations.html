Rules and Regulations

Rule 1: Code Quality

Your code should be clean, readable, and maintainable. Avoid using unnecessary variables, comments, or code blocks.

Rule 2: Functionality

Your code should correctly implement the required functionality. Make sure to test your code thoroughly before submitting it.

Rule 3: Time Limit

You have a limited amount of time to complete the test. Make sure to manage your time effectively and submit your code before the deadline.

Rule 4: Code Length

Your code should be concise and efficient. Avoid writing unnecessary code or using complex algorithms when simpler ones will suffice.

Rule 5: Code Style

Your code should follow a consistent style and formatting. Use a consistent naming convention, indentation, and spacing throughout your code.

Rule 6: No External Libraries

You are not allowed to use any external libraries or dependencies in your code. You must write your code from scratch.

Rule 7: No Cheating

You are not allowed to cheat or copy someone else's code. Any form of cheating will result in immediate disqualification.
